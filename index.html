Conceitos essenciais

Repositório (repo)
Local onde o histórico de versões do projeto é armazenado. Pode ser local (máquina) ou remoto (GitHub).
Commit
Uma "foto" do estado do projeto. Contém autor, mensagem e um hash único.
Branch
Linha de desenvolvimento isolada. Ex.: main, feature/login.
Merge
Operação que junta o histórico de duas branches.
Pull request (PR)
Mecanismo do GitHub para revisar/mesclar mudanças entre branches remotas.
Clone / Fork
Clone: cópia do repo remoto para seu ambiente. Fork: cópia do repo para sua conta no GitHub.
Remote
Referência para repositório remoto (ex.: origin).

Comandos rápidos

git clone <url>
git checkout -b feature/nova
git add .
git commit -m "mensagem"
git push origin feature/nova

# Para atualizar sua branch com main
git fetch origin
git merge origin/main

# Resolver conflitos, depois:
git add <arquivo>
git commit
git push

Fluxo recomendado (ex.: Feature Branch Workflow)

Um fluxo simples e comum para times pequenos/medianos.

Ex.: main ← PR ← feature branches → revisão → merge
1. Criar branch de feature
2. Desenvolver e commitar
3. Push e abrir PR
4. Merge

Passo a passo: Resolver conflitos de merge

  1. Atualize seu repositório local: 
    git fetch
git checkout feature/sua-branch
git pull origin main --rebase
git push origin feature/sua-branch --force-with-lease

    O rebase traz as alterações da main para sua branch (mantém histórico linear). Se preferir, use git merge origin/main.

  2. Tente mesclar (se não usou rebase): 
    git merge origin/main

    Se houver conflitos, o Git vai parar e sinalizar os arquivos.

  3. Identifique arquivos com conflito: 
    git status
  4. Abra os arquivos e resolva conflitos: 
    <<<<<< HEAD
(sua versão)
=======
(versão de origin/main)
>>>>>> origin/main

    Remova os marcadores e edite para a versão final desejada.

  5. Marque como resolvido e commit: 
    git add <arquivo-resolvido>
git commit -m "Resolve conflito: arquivo X"
  6. Atualize o remoto: 
    git push origin feature/sua-branch

    Se você usou rebase, pode ser necessário forçar o push: git push --force-with-lease.

Dicas rápidas:
  • Faça commits pequenos e mensagens descritivas.
  • Atualize sua branch com frequência para reduzir conflitos.
  • Use --force-with-lease em vez de --force para pushes após rebase.
  • Se o conflito for complexo, crie uma branch de apoio para isolar a resolução.
